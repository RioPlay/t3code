== Executive summary

T3 Code Mobile on iOS is an Expo / React Native dev-client app with five local native modules
and iOS-specific platform integrations. Android builds today but is not feature-complete.

This plan pursues *tiered product readiness* (T0→T3), not pixel or implementation parity with iOS.
The phone is a xref:research:control-plane[control plane] for delegating, monitoring, approving,
and reviewing — informed by GitHub Mobile and Linear agent-client patterns in 2026.

=== Success criteria (by tier)

*T0 (M1 pre-agent):*
* All root-stack routes reachable on Android without crash (`smoke-matrix.adoc` PASS or honest DEGRADED).
* `capabilities.ts` is sole native gate; zero `resolveNative*()!` in production paths.
* `vp check`, `vp run typecheck` pass; `npx expo-doctor` clean on native-impacting PRs.

*T1 (M1 exit — requires M0):*
* Android FCM token registers with relay; `waiting_for_approval` push opens correct thread (cold start).
* `ThreadReview` scrolls multi-file diff via `JavaScriptReviewDiffList` (no native Kotlin).
* Composer send preserves inline tokens via existing `@t3tools/shared/composerInlineTokens` (no re-extraction).
* Maestro: `smoke-launch`, `smoke-review`, `smoke-agent-push` green on CI emulator.

*T2 (M2 exit):*
* Enhanced nitro markdown: file icons, skill chips, code copy — both platforms.
* WebView+xterm terminal default on Android; RPC contract unchanged.
* Ongoing agent notification while `running` / `waiting_for_approval`.
* Thread bottom accessory bar (Files | Terminal | Review | Git).
* Tablet / foldable inner screen (≥720×600) PASS in smoke matrix + `smoke-tablet-split`.
* Maestro M2 flows green.

*T3 (M3–M4 exit):*
* Android 16: edge-to-edge insets + predictive back verified.
* Hardware keyboard shortcuts on tablet/DeX.
* Review perf gate documented in `perf-results.adoc`; Kotlin review *not* started unless gate fails.
* Optional: FGS, widget eval, Ghostty NDK spike — none block release.

*Explicitly not success criteria:*
* Matching iOS native review perf, Fabric markdown, or Ghostty terminal on phone.
* Mirror inline-chip composer (strip-mode satisfies T1/T2 composer job).
* Pixel-identical chrome or Liquid Glass.

=== Non-goals

* Pixel-identical iOS 26 glass / Liquid Glass on Android.
* Porting `t3-native-controls` or Fabric markdown to Android.
* Kotlin `T3ReviewDiffView` unless measured perf gate fails.
* Full Pierre RN diff components (`@pierre/diffs` is DOM-first — share parsers only).
* Phone-local agent execution or offline agent runs.

=== Design principles (research-validated)

. *T0 must never crash* — no `resolveNative*()!` non-null assertions.
. *Native is optional acceleration* — Bluesky small-module pattern, not monolithic Kotlin ports.
. *Same jobs, different controls* — Slack lesson; bottom nav on Android, not iOS Mail toolbar.
. *Measure before native investment* — 300ms jank gate for review; Hermes profile for markdown.
. *Agent notifications are product core* — not a polish-phase afterthought (GitHub 2026 pattern).
