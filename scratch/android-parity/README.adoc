== How to use

=== Implementing (agents + human approvals)

1. Human: paste the *Start command* from link:AGENT-LOOP.md[AGENT-LOOP.md].
2. Agent: execute **one step** from link:implementation.md[implementation.md]; stop for approval.
3. Human: merge PR → reply `continue` (or `gate pass` at step s07).
4. Repeat through **s28** (program complete). Steps s01–s28 are in link:implementation.md[implementation.md].

No weekly planning or timeline estimates — steps only.

*Human gates:* s09 (M0 staging + device), s27 (T3 device verify). *Secrets:* STAGING-RUNBOOK before s09.

=== Planning / reference

1. Read link:plan.adoc[plan.adoc] for context, tier rubric (T0→T3), and architecture.
2. Pick a milestone (*M0–M4*) and open its *companion doc* for execution detail:
+
[cols="1,2",options="header"]
|===
| Milestone | Companion doc(s)

| *M0*
| link:relay-android.adoc[relay-android.adoc], link:security.adoc[security.adoc] M0 baseline

| *M1*
| link:ux-spec.adoc[ux-spec.adoc] (strip composer, degradation labels, settings notifications),
  link:qa-strategy.adoc#release-gates[qa-strategy.adoc §Release gates] (M1)

| *M2*
| link:ux-spec.adoc[ux-spec.adoc] (accessory bar, phase indicator, ongoing notification),
  link:qa-strategy.adoc#release-gates[qa-strategy.adoc §Release gates] (M2 + tablet)

| *M3–M4*
| link:plan.adoc[plan.adoc] §M3/M4, link:qa-strategy.adoc#release-gates[qa-strategy.adoc §Release gates] (M3/M4);
  optional spikes per link:decisions.adoc[decisions.adoc]
|===
+
3. Find linked requirement IDs in link:requirements.adoc[requirements.adoc] for the milestone.
4. Reference requirement IDs (and UX-*/QA-* where applicable) in PR titles and descriptions
   (e.g. `REL-003: staging FCM push received`, `UX-021: Android notifications settings`).
5. Update link:smoke-matrix.adoc[smoke-matrix.adoc] rows to `PASS`, `DEGRADED`, or `FAIL` as features land.
6. Do not re-litigate pinned choices without a new DEC-* entry in link:decisions.adoc[decisions.adoc].

*M0 blocks M1 agent push:* do not sign off T1 control-plane rows until
link:smoke-matrix.adoc[smoke-matrix.adoc] §M0 is complete.
