== DEC-001: Android `platform` enum in relay contracts

* *Resolution:* Extend `RelayAgentAwarenessPlatform` to `Schema.Literal("ios", "android")`. Use a
  discriminated union for device registration: iOS requires `iosMajorVersion`; Android requires
  `androidSdkVersion`.
* *Date:* 2026-07-06
* *Owner:* Relay / contracts engineer
* *Revisit gate:* New mobile platform (e.g. WebPush-only client) requires new DEC entry.
* *Related:* REL-001–REL-003, SEC-010, M0 §0.1

---

== DEC-002: FCM delivery for M0–M2 (not HMS / WebPush)

* *Resolution:* Agent-awareness push on Android uses Firebase Cloud Messaging via relay worker
  (`FcmDeliveries`), mirroring APNs `push_notification` semantics. No Huawei HMS or browser WebPush
  in M0–M2 scope.
* *Date:* 2026-07-06
* *Owner:* Relay infrastructure engineer
* *Revisit gate:* Significant China-market requirement or FCM outage lasting >2 weeks with no
  mitigation.
* *Related:* REL-011–REL-030, SEC-020, SEC-060, link:relay-android.adoc[relay-android.adoc]

---

== DEC-003: Ghostty NDK demoted; WebView+xterm is GA terminal

* *Resolution:* WebView + bundled `@xterm/xterm` is the Tier-2 default terminal on Android
  indefinitely (VS Code model). Ghostty NDK is an optional M4 time-boxed spike (1 week); failure or
  slip does not block T2/T3 sign-off.
* *Date:* 2026-07-06
* *Owner:* Mobile platform lead
* *Revisit gate:* Spike succeeds *and* WebView perf gate fails on DeX/tablet hardware keyboard
  workloads.
* *Related:* TRM-002, TRM-003 (M4 optional), SEC-050–052, plan §3.1 / §4.4

---

== DEC-004: No Kotlin review diff unless perf gate fails

* *Resolution:* `JavaScriptReviewDiffList` is the permanent default on Android. Kotlin
  `T3ReviewDiffView` investment requires documented >300ms sustained scroll jank on a 10-file /
  500+ line benchmark in `perf-results.adoc`.
* *Date:* 2026-07-06
* *Owner:* Mobile performance owner
* *Revisit gate:* REV-007 gate failure with reproducible Hermes profile on target device matrix.
* *Related:* REV-001–REV-007, QA OBS-003

---

== DEC-005: Mirror composer non-blocking; strip-mode canonical

* *Resolution:* Strip-mode composer (`COMPOSER_CHIP_MODE=strip`) satisfies T1/T2 composer job.
  Mirror inline-chip editor is an optional M2+ enhancement behind
  `capabilities.composer.inlineChips.mirror` (default off). M2 sign-off does not require mirror.
* *Date:* 2026-07-06
* *Owner:* Product design + mobile engineer
* *Revisit gate:* User research shows strip-mode blocks >20% of composer workflows on Android.
* *Related:* CMP-002, CMP-003, CMP-005, link:ux-spec.adoc#strip-composer[ux-spec §strip-composer]

---

== DEC-006: `@t3tools/shared/composerInlineTokens` is canonical

* *Resolution:* Do not extract or duplicate `collectComposerInlineTokens`. Import from
  `packages/shared/src/composerInlineTokens.ts` (`@t3tools/shared/composerInlineTokens`) on all
  platforms. Android send path must produce identical token payload to iOS native editor.
* *Date:* 2026-07-06
* *Owner:* Shared packages maintainer
* *Revisit gate:* Token schema change in contracts — update shared module only, not a fork.
* *Related:* CMP-001, CMP-004

---

== DEC-007: Tier rubric sign-off (T0→T3), not outcome parity

* *Resolution:* Android parity sign-off uses tier rubric in `plan.adoc` §Executive summary. T2 is GA
  bar; T3 recommended; M4 optional. Matching iOS native implementation (Fabric, Ghostty, native
  review perf) is explicitly not a success criterion.
* *Date:* 2026-07-06
* *Owner:* Engineering lead / product
* *Revisit gate:* Executive decision to market "full iOS parity" on Android.
* *Related:* STR-001–STR-004, smoke-matrix.adoc
