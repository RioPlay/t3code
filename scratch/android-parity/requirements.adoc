== Platform capability registry

=== CAP-001: Capability module

* *File:* `apps/mobile/src/platform/capabilities.ts`
* *Must* export `platformCapabilities` resolved once at startup.
* *Must* detect native views via `getExpoViewConfig` / existing resolvers.
* *Must* support overrides:
** `EXPO_PUBLIC_FORCE_JS_REVIEW`
** `EXPO_PUBLIC_FORCE_NITRO_MARKDOWN`
** `EXPO_PUBLIC_TERMINAL_WEBVIEW`
** `EXPO_PUBLIC_COMPOSER_CHIP_MODE=strip`

=== CAP-002: No crash on missing native

* *Must not* use `resolveNativeReviewDiffView()!` or similar non-null assertions.
* *Must* render Tier 0 when resolver returns `null`.
* *Must* hide unsupported affordances with inline comment explaining why.
* Degradation copy per link:ux-spec.adoc[ux-spec.adoc] (UX-015, UX-019, UX-020).

=== CAP-003: Capability tests

* *File:* `apps/mobile/src/platform/capabilities.test.ts`
* *Must* cover forced overrides and Android-like null native resolution.
* Gate per link:qa-strategy.adoc[qa-strategy.adoc] QA-007.

== Milestone M0 — Relay & FCM

Hard prerequisite for AGT-* agent push requirements. Canonical detail:
link:relay-android.adoc[relay-android.adoc]. Security cross-refs: SEC-010–SEC-012, SEC-020,
SEC-030–SEC-033.

=== REL-001: Platform enum

* `RelayAgentAwarenessPlatform` includes `"android"`.
* *Verify:* `vp check`

=== REL-002: Android registration schema

* `RelayDeviceRegistrationRequest` decode rejects `platform: "android"` without
  `androidSdkVersion`.
* *Verify:* unit — contracts schema tests

=== REL-003: iOS registration schema

* `RelayDeviceRegistrationRequest` decode rejects `platform: "ios"` without `iosMajorVersion`.
* *Verify:* unit — contracts schema tests

=== REL-004: Android push token fields

* Android registration accepts optional `pushToken` (FCM); omits `pushToStartToken`.
* *Verify:* unit — contracts + relay HTTP test

=== REL-005: Android device persistence

* `POST /v1/mobile/devices` persists Android row with `android_sdk_version` set,
  `ios_major_version` null.
* *Verify:* integration — `Devices.test.ts`

=== REL-006: Android device list

* `GET /v1/client/devices` returns `RelayAndroidClientDeviceRecord` for Android rows.
* *Verify:* integration — API test

=== REL-007: Push token claim

* Android registration with duplicate `pushToken` claims token on new device (existing
  `Devices.register` claim behavior).
* *Verify:* unit — `Devices.test.ts`

=== REL-008: Live Activities coercion

* `preferences.liveActivitiesEnabled` coerced to `false` for Android on server ingest.
* *Verify:* unit — `Devices.register`

=== REL-009: Union typecheck

* `vp check` and `vp run typecheck` pass with union types in client-runtime + mobile.
* *Verify:* CI

=== REL-010: Mobile FCM token payload

* Mobile sends FCM token in `pushToken` field when `token.type === "android"`.
* *Verify:* mobile unit — `remoteRegistration.test.ts`

== Review diff — Tier 0 (P0, M1)

=== REV-001: JavaScriptReviewDiffList

* *File:* `apps/mobile/src/features/review/JavaScriptReviewDiffList.tsx`
* *Must* virtualize with `@legendapp/list` (not unvirtualized ScrollView).
* *Must* support row kinds: `file`, `hunk`, `line`, `notice`, `comment`.
* *Must* accept same data shape as `useNativeReviewDiffBridge` outputs.

=== REV-002: ReviewSheet branching

* *When* `!capabilities.review.native` or `FORCE_JS_REVIEW`: render JS list.
* *When* native available on iOS: unchanged native path.

=== REV-003: Review interactions

* Collapse/expand files, viewed state, line tap/long-press → comment flow.
* Pull-to-refresh, inspector scroll-to-file.

=== REV-004: Highlighting

* Visible-range Shiki via `useNativeReviewDiffHighlighting` or equivalent.
* Default engine: `javascript`.

=== REV-005: iOS dogfood

* `EXPO_PUBLIC_FORCE_JS_REVIEW=1` skips native on iOS.
* Documented in repo-root `.env.example`.

=== REV-006: Shared view-model (M2)

* *File:* `packages/shared/src/review/` (new module)
* *Must* share cache keys with `apps/web/src/lib/diffRendering.ts`.
* *Must not* import `@pierre/diffs` React components into RN.

=== REV-007: Perf gate (M3)

* Benchmark 10-file diff on Android Hermes.
* Kotlin native investment only if >300ms sustained jank documented in `perf-results.adoc`.
* Baseline capture per link:qa-strategy.adoc[qa-strategy.adoc] OBS-003.

== Agent awareness — control plane (P0–P1)

=== AGT-001: FCM registration (M1)

* Accept `type === "android"` from `getDevicePushTokenAsync()`.
* *Must not* gate registration on `canRegisterRemoteLiveActivities()`.
* *Prerequisite:* M0 complete (REL-001–REL-010).
* UX: link:ux-spec.adoc[ux-spec.adoc] UX-021, UX-022.

=== AGT-002: Notification channels (M1)

* Android channels: `agent_approval`, `agent_input`, `agent_failed`, `agent_completed`,
  `agent_running` (low).
* High importance for approval/input/failed only.
* Channel copy per link:ux-spec.adoc[ux-spec.adoc] UX-001–UX-006.

=== AGT-003: Agent-specific push copy (M1)

* Payload maps to phase: `waiting_for_approval`, `waiting_for_input`, `failed`, `completed`.
* *Must not* send generic "agent updated" for actionable phases.
* Relay FCM channel hints per link:relay-android.adoc[relay-android.adoc].

=== AGT-004: Deep links (M1)

* Reuse `useAgentNotificationNavigation`; cold start works.
* Security: link:security.adoc[security.adoc] SEC-040, SEC-041.

=== AGT-005: Ongoing notification (M2)

* Expandable/ongoing notification during `running` or `waiting_for_approval`.
* Update in place; tap deep links; dismiss does not stop agent.
* UX: link:ux-spec.adoc[ux-spec.adoc] UX-007–UX-010.

=== AGT-006: Relay payload

* Valid `RelayDeviceRegistrationRequest` for Android; passes `vp check`.
* *Prerequisite:* M0 sign-off (REL-001–REL-010) before M1 agent push work starts.
* *Must* set `platform` from `Platform.OS`; omit `iosMajorVersion` on Android.

=== AGT-007: Foreground service (M4, optional)

* User-dismissable FGS for active sessions; settings toggle.
* Not required for milestone sign-off.

== Composer (M1 fast path, M2 full)

=== CMP-001: Shared tokens (M1)

* Import `collectComposerInlineTokens` from `@t3tools/shared/composerInlineTokens`
  (`packages/shared/src/composerInlineTokens.ts`).
* *Must not* re-extract or duplicate into `packages/shared/composer`.
* Send pipeline identical token payload on Android and iOS.
* UX: link:ux-spec.adoc[ux-spec.adoc] UX-013.

=== CMP-002: Strip-mode (M1)

* `COMPOSER_CHIP_MODE=strip` shows chips in `ComposerAttachmentStrip`.
* Honest UX labeling when mirror editor not active.
* UX: link:ux-spec.adoc[ux-spec.adoc] UX-011, UX-012, UX-015.

=== CMP-003: Mirror editor (M2, optional)

* RN-only mirror layer + hidden `TextInput`.
* Chip boundary editing, paste images via `expo-paste-input`.
* iOS native editor remains default when available.
* *Non-blocking for M2 sign-off* — strip-mode (CMP-002, CMP-005) satisfies T1/T2 composer job.

=== CMP-004: Android send path (M1)

* Android `ComposerEditor` send path *must* call existing `@t3tools/shared/composerInlineTokens`
  module — same as iOS native editor and web composer.
* Token payload parity test: Android send output matches iOS reference fixture.

=== CMP-005: Strip canonical (M2)

* Mirror composer (CMP-003) is optional enhancement.
* Strip-mode + shared tokens (CMP-001, CMP-002) satisfies M2 tier sign-off (STR-003).

== Tier sign-off

Aggregate exit gates for plan rev 3 tier rubric. Mapped to milestone gates in
link:qa-strategy.adoc[qa-strategy.adoc] §Release gates.

=== STR-001: T0 — Usable (M1 pre-agent)

* All root-stack routes reachable on Android without crash (`smoke-matrix.adoc` PASS or honest
  DEGRADED).
* `capabilities.ts` is sole native gate; zero `resolveNative*()!` in production paths (CAP-001,
  CAP-002).
* `vp check`, `vp run typecheck` pass; `npx expo-doctor` clean on native-impacting PRs (QA-001,
  QA-005).

=== STR-002: T1 — Control-plane baseline (M1 exit)

* *Requires M0:* REL-001–REL-010 signed off on staging.
* Android FCM token registers with relay; `waiting_for_approval` push opens correct thread (cold
  start) — AGT-001–AGT-004, REL-010.
* `ThreadReview` scrolls multi-file diff via `JavaScriptReviewDiffList` (REV-001–REV-003).
* Composer send preserves inline tokens via `@t3tools/shared/composerInlineTokens` (CMP-001,
  CMP-004).
* Maestro: `smoke-launch`, `smoke-review`, `smoke-agent-push` green on CI emulator (QA-004).
* Notification delivery funnel visible on staging (OBS-002).

=== STR-003: T2 — Professional agent client (M2 exit)

* Enhanced nitro markdown: file icons, skill chips, code copy — both platforms (MD-001, MD-002).
* WebView+xterm terminal default on Android; RPC contract unchanged (TRM-001, TRM-002).
* Ongoing agent notification while `running` / `waiting_for_approval` (AGT-005).
* Thread bottom accessory bar — Files | Terminal | Review | Git (NAV-001).
* Tablet / foldable inner screen (≥720×600) PASS in smoke matrix + `smoke-tablet-split` (AND-003).
* Maestro M2 flows green including `smoke-composer` (QA-004).
* Mirror composer (CMP-003) *optional* — strip-mode (CMP-002, CMP-005) satisfies sign-off.

=== STR-004: T3 — Power user & platform fit (M3–M4 exit)

* Android 16: edge-to-edge insets + predictive back verified (AND-001, AND-002).
* Hardware keyboard shortcuts on tablet/DeX (TRM-005).
* Review perf gate documented in `perf-results.adoc`; Kotlin review *not* started unless REV-007
  gate fails.
* Optional M4 items (AGT-007 FGS, Ghostty NDK spike TRM-003) do not block T3 sign-off.
* T2 is GA bar; T3 recommended before broad external dogfood.

== Explicit non-requirements (resolved)

* *No* Kotlin `T3ReviewDiffView` unless REV-007 gate fails.
* *No* Kotlin `T3ComposerEditor`.
* *No* Fabric markdown on Android.
* *No* `t3-native-controls` Android port.
* *No* Pierre RN diff components.
* *No* re-extraction of `collectComposerInlineTokens` (DEC-006; use existing shared module).
* *No* mirror composer requirement for M2 sign-off (DEC-005; strip-mode canonical).
* *No* Ghostty NDK on critical path (DEC-003; WebView is T2 GA terminal).
